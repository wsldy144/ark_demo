ARKtalk 项目复盘

从游戏内私信缺口出发，设计一个可验证、可接入、可运营的 AI 攻略助手。

设计问题

玩家在游戏中遇到养成、关卡、材料问题时，需要跳出游戏寻找攻略。普通私信系统无法理解游戏上下文，普通 AI 聊天又缺少资料来源与可验证性。

解决方案

  • 将 PRTS 设计为好友列表中的特殊 AI 好友，降低接入成本。
  • 回答前先检索本地 Wiki 数据，再调用 DeepSeek 组织语言。
  • 回答结果以攻略卡片展示，包含来源、更新时间、反馈入口。
  • 结合模拟玩家档案和当前关卡上下文，生成更可执行的建议。

风险控制

  • API Key 只保存在服务端，静态页面不可访问。
  • 资料不足时返回拒答模板，避免无来源乱答。
  • 卡片标注“演示数据 / 导入骨架”，区分可信等级。
  • 用户可标记“有用 / 错误”，为后续运营校正提供入口。

验证方式

当前项目包含 Node 测试覆盖检索、RAG prompt、卡片类型、密钥保护、样式规则和导入脚本。接口冒烟验证覆盖技能、基建、材料、配队四类查询。

后续方向

  • 扩展真实 Wiki 导入脚本，从骨架导入升级为字段解析。
  • 引入 SQLite / Meilisearch，提高筛选和中文检索能力。
  • 接入游戏账号 token，替换当前模拟玩家档案。
  • 加入运营后台，查看错误反馈和高频问题。
