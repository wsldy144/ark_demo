ARKtalk / PRTS 游戏内接入说明

面向游戏内私信系统与攻略助手的 WebView 接入方案。

查看作品集说明

推荐接入方式

原型阶段采用 WebView 接入：游戏客户端打开同域聊天页，所有好友消息和 PRTS 请求都走后端 API。DeepSeek 密钥仅保存在服务端。

接入流程

  1. 游戏客户端向游戏账号服务请求一次性登录票据。
  2. 客户端打开 WebView：/index.html?token=...。
  3. 聊天后端校验票据，换取会话 ID。
  4. 玩家向好友或 PRTS 发送消息。
  5. 普通私信写入消息服务；PRTS 查询进入 Wiki 检索与 DeepSeek 调度。
  6. 前端展示文本、表情包、攻略卡片和来源更新时间。

系统结构

游戏客户端 WebView 聊天页 聊天 API 网关 PRTS Agent Wiki JSON / 搜索索引 DeepSeek API

接口表

接口 方法 用途 请求 响应
/api/session/exchange POST 游戏票据换聊天会话 { token } { sessionId, player }
/api/messages POST 发送好友私信 { sessionId, friendId, message } { messageId, createdAt }
/api/prts/chat POST PRTS 攻略查询 { message } { reply, card, sources }
/api/friends GET 获取好友列表 { sessionId } { friends }

关键边界

  • 客户端不保存 DeepSeek API Key。
  • PRTS 回答必须携带来源与更新时间。
  • 私信、举报、屏蔽、限流由聊天后端统一处理。
  • Wiki 数据先用 JSON 维护，后续可导入 SQLite 和 Meilisearch。
