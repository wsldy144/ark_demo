ARKtalk：游戏内私信与 AI 攻略助手原型

一个面向游戏内社交系统的产品策划与交互原型，核心角色为 PRTS 攻略查询终端。

项目目标

在已有好友系统但缺少私信能力的游戏中，补足一套轻量聊天系统，并将 AI 攻略助手作为特殊好友接入，使玩家可以在游戏内直接查询资料、获得结构化攻略卡片。

用户痛点

  • 玩家需要频繁跳出游戏查询 Wiki、攻略站或社区帖子。
  • 普通聊天系统只负责社交，不理解游戏上下文。
  • 攻略回答若不可验证，容易误导玩家。
  • 高难关卡和养成决策需要结合玩家已有干员，而不是给出不可执行答案。

核心功能

好友私信

好友列表、未读数、聊天记录、本地保存、搜索、置顶、免打扰、删除会话。

PRTS AI 好友

以特殊好友形式接入，支持快捷查询、加载态、结构化攻略卡片。

本地 Wiki/RAG

先检索本地 JSON 数据，再调用 DeepSeek 组织冷漠机械风回答。

可验证资料

卡片展示来源和更新时间，降低 AI 幻觉对攻略助手的伤害。

AI 架构

  1. 玩家在 PRTS 私信窗口输入问题或点击快捷查询。
  2. 服务端在本地 Wiki JSON 中检索干员、材料、账号上下文。
  3. 检索结果被整理为 RAG 参考资料。
  4. DeepSeek 根据系统 Prompt 和参考资料生成 PRTS 风格回答。
  5. 前端同时展示文本回复与结构化攻略卡片。

当前演示数据

本原型使用 5 个干员 JSON、1 个材料 JSON 和 1 个模拟玩家档案，验证数据维护方式与卡片生成链路。后续可导入 SQLite、Meilisearch 或真实 Wiki 爬虫索引。

迭代路线

  • 阶段 1：扩展干员、材料、关卡数据结构。
  • 阶段 2：加入聊天服务端持久化和真实好友 API。
  • 阶段 3：接入游戏账号 token，支持 WebView 内鉴权。
  • 阶段 4：建立 Meilisearch 索引，提高中文检索质量。
  • 阶段 5：加入反馈系统，持续修正错误攻略回答。
